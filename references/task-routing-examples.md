# Task Routing Examples

> 目标：把主控 Claw / mini nini 的职责边界翻译成可直接参考的任务路由表。

## 1. 家庭 / 私人类

| 任务类型 | 默认入口 | 是否需要主控 | 说明 |
|---|---|---|---|
| B 超 / 孕检解读 | 主控 Claw（微信） | 是 | 属于家庭健康与解释型任务，应由主控负责判断与说明 |
| 房产看房 / 生活决策 | 主控 Claw（微信） | 是 | 涉及信息整合、判断与建议，不建议直接交执行层 |
| 果果作业 / 学校事务 | 主控 Claw（微信） | 是 | 以理解需求、组织表达、给建议为主 |
| 私人提醒 / 家庭安排 | 主控 Claw（微信） | 视情况 | 轻任务可直接处理，若涉及定时执行可转交 mini |

## 2. 研究 / 决策类

| 任务类型 | 默认入口 | 是否需要主控 | 说明 |
|---|---|---|---|
| GitHub 项目分析 | 主控 Claw（Telegram） | 是 | 先由主控分析，再决定是否让 mini 拉仓库/跑验证 |
| 架构设计 / 方案讨论 | 主控 Claw（Telegram） | 是 | 典型主控任务 |
| 长文 / 观点分析 | 主控 Claw（Telegram/微信） | 是 | 需要总结、判断、提炼观点 |
| 多来源研究任务 | 主控 Claw（Telegram） | 是 | 主控负责研究路线，mini 可做补充执行 |

## 3. 执行 / 落地类

| 任务类型 | 默认入口 | 是否可直达 mini | 说明 |
|---|---|---|---|
| 跑脚本 / 跑命令 | mini nini | 是 | 明确、低歧义、输入输出清楚时可直接交 mini |
| 整理目录 / 文件归档 | mini nini | 是 | 典型执行层任务 |
| 批量数据处理 | mini nini | 是 | 数据加工与本地执行优先在 mini |
| 知识库文件更新 / 索引维护 | mini nini | 是 | 作为知识库宿主，mini 更适合承担 |
| 长时后台任务 | mini nini | 是 | 执行层长期承载 |

## 4. 系统 / 运维类

| 任务类型 | 默认入口 | 是否需要主控 | 说明 |
|---|---|---|---|
| OpenClaw 配置 / 升级 / 排障 | 主控 Claw（Telegram） | 是 | 主控负责判断、排障思路与风险控制；必要时派 mini 执行 |
| 插件审计 / 安装 | 主控 Claw（Telegram） | 是 | 审计与决策在主控，安装执行可交 mini |
| 定时任务 / cron 设计 | 主控 Claw | 是 | 主控决定是否创建及其策略 |
| 机器本地状态检查 | mini nini | 可直接 | 如 CPU / 磁盘 / 服务状态等，本地节点直接处理更合适 |

## 5. 协作闭环类

| 场景 | 推荐流程 |
|---|---|
| 复杂任务需要本地执行验证 | 主控分析 → 派发 mini → mini 回报 → 主控总结 |
| 数据在 mini、本地要产出结论 | 主控提出问题 → mini 提供数据/执行结果 → 主控形成最终判断 |
| 低歧义纯执行任务 | 直接给 mini |
| 高歧义或需要统一口径的任务 | 必须先给主控 |

## 6. 简化原则
- 默认：复杂任务先给主控
- 默认：执行型任务可给 mini
- 数据和知识库主库在 mini
- 高层结论、长期判断、任务状态在主控
- 若任务从轻变重，升级到主控
- 若任务从分析进入执行，交给 mini 落地
