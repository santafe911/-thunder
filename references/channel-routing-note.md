# Channel Routing Note

> 目标：让不同入口承担不同气质和任务类型，减少上下文串味。

## 微信（私人 / 家庭入口）
优先承接：
- B 超 / 孕检 / 家庭健康类
- 房产看房 / 生活决策类
- 果果作业 / 学习 / 学校事务
- 私人提醒、家庭安排、日常琐事

定位：
- 家庭生活助理
- 私人事务入口
- 果果事项入口

## Telegram（工作台 / 控制台入口）
优先承接：
- OpenClaw 配置 / 升级 / 排障
- 插件审计 / 安装
- GitHub 项目分析
- Agent 系统设计
- 长篇研究、方案、架构讨论

定位：
- 技术工作台
- 研究和项目主入口
- 系统控制面板

## 简化原则
- 微信优先处理与“人 / 家 / 生活”直接相关的任务
- Telegram 优先处理与“系统 / 项目 / 研究 / 搭建”直接相关的任务
- 若任务从轻变重，可从微信切到 Telegram 继续
- 若任务从技术延伸到生活执行，可从 Telegram 回到微信承接
