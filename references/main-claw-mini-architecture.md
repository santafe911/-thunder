# Main Claw / mini nini Architecture

> 总原则：所有复杂任务先进入主控；主控决定是否派发给 mini；mini 负责执行、沉淀、回报。

## 1. 角色定位

### 主控 Claw
定位：
- 总调度
- 总参谋
- 统一入口
- 决策层

负责：
- 理解需求
- 拆解任务
- 判断优先级
- 做风险与边界判断
- 决定是否派发给 mini
- 汇总 mini 的结果
- 向 Leo 输出最终结论与建议

### mini nini
定位：
- 执行节点
- 本地操作员
- 数据层与知识库宿主
- 长时任务承载点

负责：
- 本地代码与脚本执行
- 文件处理与整理
- 数据缓存、数据加工、数据沉淀
- 工作知识库维护
- 定时任务与长时运行任务
- 按任务要求回报结果

## 2. 路由原则

### 必须先进主控的任务
- 复杂研究与分析
- 方案设计与路线判断
- 多步任务与跨域任务
- 需要风险权衡的任务
- 需要做优先级取舍的任务
- 需要最终决策口径统一的任务

### 可直接给 mini 的任务
- 明确的执行型任务
- 已有清晰输入输出的本地处理任务
- 跑脚本 / 查状态 / 整理目录 / 导出文件
- 更新本地知识库 / 数据库 / 缓存
- 长时运行但低歧义的任务

## 3. 数据与知识边界

### mini 保存
- 原始数据
- 工作数据
- 本地知识库主库
- 中间产物
- 日志 / 缓存 / 执行结果文件

### 主控保留
- 抽象结论
- 路由规则
- 任务状态
- 高层记忆
- 对 Leo 有意义的长期判断

原则：
- 全量资产放 mini
- 抽象认知放主控
- 不做无脑全量同步
- 优先同步摘要、结论、索引与 handoff

## 4. mini 回报格式
mini 完成任务后，优先按以下结构回报：
1. 做了什么
2. 产物在哪
3. 结果如何
4. 是否有异常
5. 是否需要主控继续判断

## 5. 系统原则
- 主控权在主控 Claw，不在 mini
- mini 不负责定义方向，mini 负责把方向落地
- 复杂任务默认先经主控
- mini 是执行层，不是第二个老板
- 自动化打通应建立在职责清晰之后
