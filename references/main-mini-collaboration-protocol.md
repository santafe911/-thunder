# Main Claw / mini nini Collaboration Protocol

> 目标：建立主控与执行层之间轻量、稳定、可审计的任务转派与结果回报协议。

## 1. 主控派单模板

```text
[任务标题]

任务目标：
- 这次要完成什么

输入材料：
- 文件 / 路径 / 链接 / 上下文

执行边界：
- 可以做什么
- 不可以做什么
- 是否允许改文件 / 跑命令 / 长时任务

输出要求：
- 最终要交付什么
- 文件 / 摘要 / 日志 / 截图 / JSON 等

验证要求：
- 如何证明任务完成
- 需要回显哪些状态 / 输出 / 文件存在性

回报要求：
- 完成后是否需要主控继续判断
- 若遇到异常，优先回报什么
```

## 2. mini 回报模板

```text
[任务回报]

已完成：
- 做了什么

产物位置：
- 文件在哪
- 日志在哪
- 输出在哪

结果摘要：
- 结果如何
- 关键发现是什么

异常 / 风险：
- 是否有异常
- 哪些地方不确定
- 是否触碰边界

是否需要主控继续判断：
- 是 / 否
- 如果需要，请说明下一步需要判断什么
```

## 3. 状态枚举

建议统一使用以下状态：
- `todo` — 待执行
- `doing` — 执行中
- `done` — 已完成
- `blocked` — 被阻塞
- `need-decision` — 需要主控继续判断

## 4. 协作原则
- 主控定义任务，mini 执行任务
- 主控解释结果，mini 回传结果
- 复杂任务默认先经主控
- mini 不负责定义方向，只负责落地方向
- 原始产物留在 mini，摘要与结论回主控
- 不做无脑全量同步，优先同步状态、索引、结论

## 5. 推荐使用场景

### 主控 → mini
- 跑脚本
- 本地数据处理
- 文件整理
- 知识库维护
- 状态检查
- 长时任务执行

### mini → 主控
- 结果摘要
- 异常与阻塞
- 关键结论
- 是否需要进一步判断
